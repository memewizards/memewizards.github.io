Detailed Description

this is a generic class that takes fonctors as undo and redo actions. It just executes them when required by Qt Note that QUndoStack actually executes redo() when we push the undoCommand to the stack This is bad for us because we execute the command as we construct the undo Function. So to prevent it to be executed twice, there is a small hack in this command that prevent redoing if it has not been undone before.
